## Durable Beliefs

- **Practical coding beats theory-first.** The best preparation for CS is becoming familiar with the practical side — tools, debugging, and writing real code — not studying CS theory like relational database math or cryptography before you need it. You'll forget it anyway. [Note #128, 2026-06-28]
- **C is the right first language, not Python.** Harvard CS50x is the best intro CS course because it starts with C, which doesn't hold your hand and forces you to learn memory management early. Python as a first language hides pointers and memory, creating confusion later when concepts like pass-by-value vs pass-by-reference appear in other languages. [Note #128, 2026-06-28]
- **Learn debugging tools properly.** IDE debuggers (breakpoints, variable inspection) are better than print statements most of the time. If your course forces terminal-only exams, learn GDB — it's powerful but painful to use without the TUI layout. Valgrind is niche but essential for C memory debugging. [Note #128, 2026-06-28]
- **Vim is worth learning** if you'll face terminal-only programming exams. The keybinds are bizarre but efficient once internalized, and every major IDE has a vim keybinds plugin. [Note #128, 2026-06-28]
- **ChatGPT (GPT-4) is 100% worth paying for** in CS education — treat it as a personal tutor that explains things, not a source of unpaid labour. [Note #128, 2026-06-28]
- **NUS Security focus area isn't well taught.** Better off getting external certs like OSCP rather than relying on NUS security modules. [Note #128, 2026-06-28]
